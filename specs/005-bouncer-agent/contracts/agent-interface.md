# Agent Interface Contract: Bouncer Agent

**Version**: 1.0  
**Date**: 2026-03-07  
**Feature**: [spec.md](../spec.md)

## Overview

This document defines the interface contract for the Bouncer Agent node in the LangGraph pipeline. It specifies the expected inputs, outputs, preconditions, and postconditions that other agents and the graph controller must adhere to.

---

## 1. Agent Signature

```python
def bouncer_agent(state: GraphState) -> dict:
    """
    Bouncer Agent: Classifies customer intent and routes to appropriate specialist.
    
    This is a single-turn agent that reads the authenticated user's tier,
    classifies their intent from conversation history, and routes them to the
    correct Specialist Agent.
    
    Args:
        state (GraphState): Current graph state with authenticated user and history
        
    Returns:
        dict: State updates including customer_tier, customer_intent, current_agent,
              and handoff message appended to messages
              
    Preconditions:
        - state['is_authenticated'] == True
        - state['verified_user'] is not None
        - state['verified_user'].tier in ["standard", "premium", "vip"]
        - state['messages'] contains conversation history
        
    Postconditions:
        - state['customer_tier'] is set to tier value
        - state['customer_intent'] is set to classified intent
        - state['current_agent'] is set to target specialist name
        - Handoff message is appended to state['messages']
    """
```

---

## 2. Input Contract

### 2.1 Required State Fields

| Field | Type | Constraint | Description |
|-------|------|------------|-------------|
| `is_authenticated` | `bool` | **MUST be `True`** | Enforced by conditional edge before Bouncer |
| `verified_user` | `User` | **MUST NOT be `None`** | Set by Greeter after successful authentication |
| `verified_user.tier` | `str` | **MUST be one of**: `"standard"`, `"premium"`, `"vip"` | Customer tier from database |
| `messages` | `List[BaseMessage]` | **MUST NOT be empty** | Conversation history for intent classification |

### 2.2 Precondition Enforcement

**Responsibility**: LangGraph conditional edge logic

**Before Bouncer Invocation**:
```python
def should_route_to_bouncer(state: GraphState) -> str:
    """Conditional edge from Greeter to Bouncer."""
    if state.get("is_authenticated"):
        return "bouncer"
    else:
        return "greeter"  # Loop back if auth failed
```

**Bouncer MUST assume**:
- Preconditions are met (no need to re-check authentication)
- `verified_user` is a valid User object from database
- Conversation history exists (at minimum: greeting + auth turns)

---

## 3. Output Contract

### 3.1 Required State Updates

| Field | Type | Value | Description |
|-------|------|-------|-------------|
| `customer_tier` | `str` | `"standard"` \| `"premium"` \| `"vip"` | Tier copied from `verified_user.tier` |
| `customer_intent` | `str` | One of 5 intents or `"general_inquiry"` | Classified intent (with confidence fallback) |
| `current_agent` | `str` | `"specialist_standard"` \| `"specialist_premium"` \| `"specialist_vip"` | Target specialist for routing |
| `messages` | `List[BaseMessage]` | Original + new `AIMessage` | Handoff message appended |

### 3.2 Intent Values

**Guaranteed**: `customer_intent` will be one of:
- `"account_balance"`
- `"transaction_history"`
- `"fund_transfer"`
- `"lost_card"`
- `"general_inquiry"`

**Note**: If LLM confidence < 0.5, intent is automatically set to `"general_inquiry"`.

### 3.3 Routing Values

**Guaranteed**: `current_agent` will be one of:
- `"specialist_standard"` (if tier == `"standard"`)
- `"specialist_premium"` (if tier == `"premium"`)
- `"specialist_vip"` (if tier == `"vip"`)
- `"specialist_standard"` (default if tier unrecognized)

### 3.4 Handoff Message Format

**Type**: `AIMessage`

**Content**:
- One sentence
- Warm and professional tone
- Does NOT reveal customer tier
- Passes guardrails output check

**Examples** (generated by LLM):
- "Connecting you to your dedicated advisor..."
- "Routing you to our support team..."
- "One moment while I transfer you to a specialist..."

## 4. Error Handling Contract

### 4.1 Defensive Behavior

| Error Condition | Bouncer Response |
|-----------------|------------------|
| `verified_user.tier` not in routing map | Default to `"specialist_standard"` |
| LLM intent classification fails | Default to `"general_inquiry"` |
| LLM confidence < 0.5 | Override to `"general_inquiry"` |
| Input message fails guardrails | Return safe response; **do NOT route** (early exit) |
| Output message fails guardrails | Use sanitised response |

### 4.2 No-Throw Guarantee

**Contract**: `bouncer_agent()` MUST NOT raise exceptions for expected error conditions.

**Rationale**: LangGraph pipeline should continue even if Bouncer encounters issues. Fallback to safe defaults ensures graceful degradation.

**Exception**: Unrecoverable errors (e.g., OpenAI API key missing) may propagate as system-level failures.

---

## 5. Side Effects

### 5.1 External API Calls

| Service | Purpose | Failure Handling |
|---------|---------|------------------|
| OpenAI API | Intent classification + handoff message generation | Fallback to defaults (general_inquiry) |
| Guardrails | Input/output safety checks | Use safe_response or sanitised_response |

### 5.2 State Mutations

**Immutability**: Bouncer does NOT mutate existing state fields (except appending to `messages`).

**Isolation**: Bouncer does NOT call other agents directly. Routing is handled by LangGraph based on `current_agent` value.

### 6.1 Latency Targets

| Metric | Target | Rationale |
|--------|--------|-----------|
| Average latency | < 500ms | Single LLM call + guardrails |
| P95 latency | < 1000ms | Acceptable for routing operation |
| P99 latency | < 2000ms | Includes retry logic |

### 6.2 Resource Usage

- **LLM Calls**: Exactly 2 per invocation (1 for intent, 1 for handoff message)
- **Memory**: < 10 MB (conversation history + model outputs)
- **Tokens**: ~200 output tokens (intent + handoff message)

## 8. Integration Contract

### 8.1 Upstream Agent (Greeter)

**Greeter Responsibilities**:
- Set `is_authenticated = True` before handoff
- Set `verified_user` with valid User object
- Ensure `verified_user.tier` is one of the three supported tiers

**Greeter MUST NOT**:
- Set `customer_tier` or `customer_intent` (Bouncer's job)
- Set `current_agent` to specialist (Bouncer's job)

### 8.2 Downstream Agents (Specialists)

**Specialist Expectations**:
- `customer_tier` is available and valid
- `customer_intent` is available and valid
- `messages` includes Bouncer's handoff message

**Specialist MUST NOT**:
- Re-authenticate the user (trust Bouncer's routing)
- Re-classify intent (may refine, but Bouncer's classification is authoritative)

### 8.3 Graph Controller

**LangGraph Edge Conditions**:

```python
# After Greeter
def route_from_greeter(state: GraphState) -> str:
    if state.get("is_authenticated"):
        return "bouncer"
    elif state.get("conversation_ended"):
        return END
    else:
        return "greeter"

# After Bouncer
def route_from_bouncer(state: GraphState) -> str:
    # Bouncer always routes to the specialist it set in current_agent
    return state["current_agent"]  # One of: specialist_standard, specialist_premium, specialist_vip
```
